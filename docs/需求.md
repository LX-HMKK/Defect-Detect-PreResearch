# 项目目标:
- 探索并分析现有无监督异常检测算法在工业场景下的适应性与可用性

# 项目背景：
## 1. 工业视觉检测现状
- 工业流水线工艺成熟、生产稳定
- 异常/缺陷样本占比极低，采集与标注成本高
- 缺陷形态多样，且存在未知缺陷
## 2. 传统方法的局限性
- 有监督检测方法高度依赖大量已标注缺陷样本
- 对训练集中出现的缺陷类型依赖强
- 面对新缺陷或分布变化时，泛化能力与稳定性不足
## 3. 无监督异常检测的优势
- 仅使用正常样本进行训练，符合工业数据分布特点
- 通过学习正常产品的外观、纹理和结构分布，在推理阶段根据与“正常模式”的偏离程度实现异常检测与异常定位
- 对未知缺陷具有更好的适应性与工程实用价值


# 项目要求：
## 1. 方法： 无监督异常检测方法调研与复现
系统性调研并分析主流无监督工业异常检测技术路线，包括但不限于以下方向：
- 基于重构的方法（如 AutoEncoder、VAE 等）
- 基于特征建模的方法（如 Patch-level 特征分布建模、统计距离度量）
- 基于自监督表示学习的方法
- 基于预训练大模型或基础模型的方法        

在每一类代表性方向中，至少复现并实现一种典型无监督异常检测算法，并结合工业异常检测任务，从检测精度、定位能力、稳定性及计算复杂度等角度，对不同方法的特点、适用场景及优劣势进行分析与总结(若有能力可展开适当的方法创新)。

## 2. 训练数据约束条件
- 模型训练阶段仅允许使用正常样本
- 考虑到项目初期可调试样本数量有限，用于训练的正常样本数量不超过 150 张
- 需分析小样本条件下不同方法的性能变化及鲁棒性表现

## 3.实验验证与性能分析
- 在公开工业异常检测数据集和提供的真实数据集上完成异常检测与异常定位实验
- 图像级异常检测指标：AUROC，AUPR，像素级异常定位指标:  Pixel-level AUROC,PRO
## 4、界面设计
- 设计一个UI界面，训练样本选择，模型的训练推理（训练过程和结果展示），需要呈现在这个界面。


# 项目验收：
1. 企业需要的是一份能为未来提供参考的综述，详细对比所选算法的优劣势及工业场景适用性
2. 不会使用项目中搭建的 UI 界面，只是为了满足课题验收和可视化的需要
3. 代码仓库：需要提交复现和测试的代码，主要为了证明实验的真实性和可追溯性
